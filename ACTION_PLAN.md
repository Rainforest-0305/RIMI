# 공시알리미 (gongsi-alert) — 액션플랜

> DART/SEC 공시 AI 알림 서비스. 관심종목/키워드 등록 → 신규 공시 감지 →
> 핵심 3줄 요약 + 영향 태그 → 텔레그램/웹 알림.
> **매매추천 없음. 순수 정보·요약·분류.** (자문규제 밖 유지)

작성 관점: pm · backend · bizops · growth 종합. 실행 환경은 노트북 1대이므로
**동시 작업자 ≤2, 무거운 연산 없음(대부분 IO/코딩)** 을 전제로 스코프를 잡았다.

## 1. 2주 마일스톤 (주차별)

### Week 1 — "작동하는 코어" (backend 주도, pm 검수)
| 일 | 작업 | 완료 정의 |
|---|---|---|
| D1 | 스캐폴드 + DART list.json 폴링 + corp_code 매핑 | `--demo` 실공시 출력 ✅(완료) |
| D2 | 규칙기반 분류(영향 태그) + 3줄 요약 스텁 + LLM 인터페이스 격리 | 태그 15종, `set_llm_hook` ✅(완료) |
| D3 | 중복방지(seen 원자적 저장) + 알림(텔레그램 테스트채널/콘솔 폴백) | 재실행 시 0건 재알림 ✅(완료) |
| D4 | 랜딩 1페이지(관심종목 등록 폼 목업) + 규제 고지문 | index.html ✅(완료) |
| D5 | LLM 요약 훅 실장(제목+공시유형 → Claude 3줄 요약), 품질 스팟체크 20건 | 요약 오독률 <10% |
| D6-7 | 상주 폴링 안정화(유량/에러 fail-open), 관심종목 CRUD를 JSON→SQLite | 24h 무중단 폴링 로그 |

### Week 2 — "유저가 쓸 수 있는 것" (backend+growth 2인 동시, 상한 준수)
| 일 | 작업 | 완료 정의 |
|---|---|---|
| D8 | 텔레그램 봇 명령화(/add 005930, /list, /del) — **본인 테스트 채널 내에서만** | 봇으로 종목 등록/알림 왕복 |
| D9 | 웹 등록 폼 → 백엔드 연동(FastAPI 최소 API + SQLite) | 폼 제출 → DB 저장 → 알림 수신 |
| D10 | 알림 히스토리/피드 페이지(최근 공시 타임라인) | 웹에서 지난 알림 열람 |
| D11 | 온보딩 카피 + 규제 고지 최종화 + 개인정보 최소수집 점검 | 약관/개인정보 문구 확정 |
| D12 | **비공개 베타**: 지인/본인 계정 5~10명(Partner 승인 후에만 외부 발송) | 실사용 피드백 5건 |
| D13 | KPI 계측(알림 수·클릭·유지) 대시보드 최소판 | 지표 자동 집계 |
| D14 | 회고 + SEC 확장/과금 실험 설계 확정 | Week3 백로그 |

> 부하 관리: 폴링은 5분 주기·종목당 0.3초 간격·요약 LLM 호출은 신규 공시당 1회.
> 동시 작업은 backend 1 + growth 1 최대. LLM 대량 배치·크롤링 등 무거운 작업 금지.

## 2. 기술 스택 · 아키텍처

```
[관심종목 DB]        [DART OpenAPI list.json]        [SEC EDGAR (확장)]
   (SQLite)      ┌────────── 폴링(5분) ──────────┐
       │         │  dart_poll.fetch_disclosures  │
       ▼         ▼                               ▼
  ┌─────────────────────────────────────────────────┐
  │  main.poll_once  — 신규만 필터(seen.json, 원자적)  │
  └───────────────┬─────────────────────────────────┘
                  ▼
      summarize.summarize()
        ├ classify()      규칙기반 영향 태그(결정적)
        └ llm_summarize() 3줄 요약  ← set_llm_hook()로 Claude 주입
                  ▼
      notify_alert.send()
        ├ 텔레그램(테스트 채널 전용)   ├ 웹 피드
        └ 콘솔/파일 폴백(감사로그)     (fail-open: 알림 실패해도 폴링 지속)
```

- **언어/런타임**: Python 3.11 (기존 트레이딩 자산과 동일 — requests, python-dotenv 재사용)
- **저장소**: MVP는 JSON(watchlist/seen) → Week2에 SQLite(관심종목·유저·알림 이력)
- **웹**: 정적 랜딩(현재) → FastAPI + 단일 페이지(등록/피드). 무거운 프레임워크 지양.
- **요약 LLM**: `summarize.set_llm_hook()` 한 지점만 교체. 공시 제목·유형(원문 X)
  기반 저비용 요약부터 시작(토큰·부하 절감), 필요 시 원문 일부 확장.
- **알림**: 텔레그램(notify.py tg 패턴) — **테스트 채널로만**. 이메일/웹푸시는 후속.
- **배포**: 초기엔 노트북 상주(schtasks 등록). 트레이딩 시스템과 **키만 공유(읽기)**,
  프로세스·포트·상태파일 완전 격리(`ventures/gongsi-alert` 전용 폴더).

**재사용 자산**: DART_API_KEY·텔레그램 토큰(kis-trading/.env 읽기전용),
dart_fundamental.py의 corp_map 패턴, notify.py의 tg 패턴, 멀티에이전트 요약 로직.

## 3. 초기 10~100명 유저 확보 채널

우선순위 = 이미 "공시를 실시간으로 알고 싶은 사람"이 모인 곳.
1. **투자 커뮤니티 시딩**: 네이버 종목토론방/카페, 디시 주식갤, 클리앙/뽐뿌 재테크,
   Reddit r/stocks(SEC 확장 시). "관심종목 공시를 3줄로" 데모 GIF 1개.
2. **텔레그램 투자방**: 국내 종목 리딩방·스터디방에 무료 공개 알림봇으로 소개
   (매매신호 아님 = 리딩방과 비경쟁 → 오히려 보완재로 포지셔닝).
3. **X(트위터)**: 특정 종목 공시 뜰 때 자동 3줄 요약을 공개 계정으로 게시(허가된
   공개정보), 프로필에 등록 링크. "공시 봇" 니치 팔로워 확보.
4. **에버그린 콘텐츠**: "유상증자 공시 읽는 법", "자사주 소각 vs 취득 차이" 같은
   해설 글 → SEO/커뮤니티 유입 → 알림 등록 CTA.
5. **개인 네트워크/지인 베타**: 첫 10명은 손으로 모은다(정성 피드백 목적).

> 성장 원칙: 처음엔 **무료·무마찰(종목코드만 입력)**. 바이럴 훅 =
> "이 공시 3줄 요약" 스크린샷 공유. 스팸/무단 대량발송 금지(Partner 승인 전 외부발송 X).

## 4. 첫 과금 실험 (가설 · 가격)

- **핵심 가설**: "관심종목이 많고 공시에 민감한 액티브 투자자는 종목 수 제한 해제와
  즉시성(딜레이 없는 알림)에 월 소액을 낸다."
- **프리미엄 경계(Free → Paid)**:
  | 구분 | Free | Pro (가설) |
  |---|---|---|
  | 관심종목 | 3~5종목 | 무제한 |
  | 알림 지연 | 5~10분 폴링 | 우선(1~2분) |
  | 요약 | 3줄 규칙+기본 | 유형별 심화 3줄 + 히스토리 검색 |
  | 채널 | 웹 | 텔레그램+웹+이메일 |
  | SEC(미국) | ✕ | ○ |
- **가격 실험**: 월 3,900원 / 5,900원 / 9,900원 3구간 A/B(랜딩 가격 노출 → 클릭률).
  결제 연동은 **Partner 승인 후** (Stripe/토스). 그 전엔 "관심 등록(waitlist)"으로 수요 측정.
- **성공 기준**: waitlist→유료전환 의향 클릭 CTR ≥ 10%, 무료 유저 중 종목한도 도달 비율 ≥ 30%.

## 6. 규제 · 약관 체크 (론칭 전 필수)

- **투자권유 아님 명시**: 모든 알림·페이지에 "정보 제공이며 투자권유가 아님" 고지.
  → 이미 알림 메시지 말미·랜딩 하단에 삽입 완료.
- **원문 대체 아님**: "요약은 공시 원문을 대체하지 않으며, 판단·책임은 이용자에게".
  → 각 알림에 원문(DART) 링크 필수 첨부(구현됨).
- **매매신호 배제**: 매수/매도/목표가/의견 문구 금지 — 분류는 사실 태그(유상증자·실적 등)만.
- **개인정보 최소수집**: 알림에 필요한 식별자(텔레그램 chat_id 또는 이메일)만.
  민감정보·계좌정보 수집 금지. 수집 항목·목적·보관기간 고지, 탈퇴 시 즉시 삭제.
- **데이터 출처/저작권**: DART/SEC 공개데이터. 원문 재배포 대신 링크+요약. 출처 표기.
- **자문업 경계**: 유사투자자문/투자자문업 등록 대상이 되지 않도록 "특정인 대상 개별
  종목 매매판단 제공"을 하지 않음(불특정 다수에 사실 정보 요약만). 유료화 시 문구 재검토.
- **발송 규제**: 광고성 정보 발송 시 수신동의·야간발송 제한 등 준수(정보성 알림과 구분).

## 7. 리스크 · 다음 확장

**리스크**
- DART 유량제한/스키마 변경 → 폴링 실패. 대응: fail-open(폴링 지속) + 재시도 + 상태코드 감시.
- 요약 오독(LLM 환각) → 신뢰 훼손. 대응: 분류는 결정적 규칙, 요약엔 항상 "원문 확인" 고지 + 원문 링크.
- 알림 지연/누락 → 핵심 가치 훼손. 대응: seen 원자적 저장, 폴링 성공률 KPI 감시.
- **트레이딩 시스템 오염 금지**: 키는 읽기만, 프로세스·상태·텔레그램 채널 완전 격리.
  외부 실유저 발송·과금·도메인은 Partner 승인 전 금지.
- 규제 리스크: 유료화·개별 종목 코멘트 확장 시 자문업 경계 재검토.

**다음 확장 (우선순위)**
1. **요약 품질 승격**: 제목 기반 → 공시 원문 핵심문단 기반 3줄(정확도↑).
2. **SEC(미국) 확장**: sec_fundamental.py의 CIK 매핑·EDGAR 패턴 재사용(키 불필요),
   8-K/10-Q 등 폼 타입 태깅. 글로벌 유저·X 채널과 시너지.
3. **키워드/섹터 알림**: 종목뿐 아니라 "전 종목 유상증자", 섹터/테마 단위 구독.
4. **웹 대시보드 고도화**: 알림 히스토리·필터·검색.
5. **계좌연동(후속·신중)**: 보유종목 자동 관심등록. 단, 자문규제·보안 검토 선행.
6. **과금·결제 연동**: 수요 검증(waitlist CTR) 통과 후 Partner 승인 하에 도입.
